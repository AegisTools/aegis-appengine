What is Markdown?

Markdown is a set of rules that let you format text without a lot of completely meaningless symbols. In general markdown looks... ok as text. But once the system formats it, it looks better.

Basic Formatting

There are a few basic things that you can do for emphasis. Wrapping something in single asterisks makes it italic; two asterisks makes it bold; three makes it bold and italic. If you really don't like asterisks you can use underscores (_). You can use backticks (`) for a different type of emphasis, but this is usually used for code.

Headers

There are a few ways to do headers. Underlining something with at least 3 equals signs (=) makes it the largest header possible (called "header 1"). Underlining with dashes (-) will make it the next largest header ("header 2").

You can also make headers by putting pound symbols (#) at the beginning of the line. One symbol for header 1, two for header 2, and so on. They go up to header 6

Lists

There are a few kinds of lists that you can do. Bulleted lists are done by putting a dash and a space (- ) before the text you want to be a list. You can indent lists by putting two spaces.

You can do a numbered list by using a number followed by a period and a space (1. ). The actual number is actually ignored, so don't worry about getting the numbering correct.

You can also put one kind of list inside another, putting a bullet list inside a numbered list, etc. You can not mix styles within a list (e.g. make a few things numbered, then a few bulleted). Also, a single line break is ignored. If you want to stop your list and start a new one you need more than one newline.

Blocks

A single newline is generally ignored in a block of text, but two newlines (leaving a space) will force the start of a new paragraph. If you want a little bit more space and emphasis between paragraphs you can put a line all by itself with at least three dashes (---) to create a horizontal line.

Indenting a whole paragraph with at least four spaces creates a block of content, but this is intended for code formatting. Indenting a paragraph with a greater-than symbol (>) creates a quoted block. This indicates that you are directly quoting something somebdy said. This is particularly useful when you want to put answers directly after questions.

Links

URLs are auto-detected. You don't need to do anything to make them show up properly, but you can replace the text. Put the text in brackets ([]) and the URL in parentheses (()) right afterwards. You can put the URL anywhere else, but the syntax is hard to explain. See the examples below.

Images

You can link to images as well. The syntax is identical to links, but you put an exclamation point (!) right before it.
